# Threat vs Vulnerability Analysis

| No | Scenario  | Category  | Reasoning |
|----|---------------------|-------------------|-----------|
| 1 | An employee receives an email from “HR Department” asking to verify login details via a link. | Threat | Phishing – external malicious attempt to steal credentials. |
| 2 | A company server is running Windows Server 2012 with no security updates. | Vulnerability | Outdated software – exploitable weakness. |
| 3 | A developer uploads API keys to a public GitHub repository. | Vulnerability | Sensitive information exposure. |
| 4 | A hacker uses a botnet to flood a website. | Threat | DDoS attack – active external attack. |
| 5 | Firewall is misconfigured to allow all inbound traffic. | Vulnerability | Weak configuration. |
| 6 | Insider copies confidential data to personal USB. | Threat | Insider malicious activity. |
| 7 | Password policy allows “12345” as password. | Vulnerability | Weak access control policy. |
| 8 | Cybercriminals send ransomware via email attachments. | Threat | Ransomware attack. |
| 9 | Website doesn’t validate input, allowing SQL commands. | Vulnerability | SQL Injection risk. |
| 10 | Fake job ads used to steal personal data. | Threat | Social engineering attack. |
